## Framework-Specific Considerations

### Mantine Projects - Component Usage Guidelines

**Card Component and Subcomponents**:
1. **Card.Section Usage**: When using Mantine Card components, Card.Section should be properly imported and used
   ```javascript
   // ‚úÖ CORRECT - Import Card and use Card.Section
   import { Card, Text, Image, Group } from '@mantine/core';
   
   <Card shadow="sm" padding="lg" radius="md" withBorder>
     <Card.Section>
       <Image src="image.jpg" height={160} alt="Description" />
     </Card.Section>
     
     <Group justify="space-between" mt="md" mb="xs">
       <Text fw={500}>Card Title</Text>
     </Group>
     
     <Text size="sm" c="dimmed">
       Card description content
     </Text>
   </Card>
   
   // ‚ùå WRONG - Don't use 'section' as a separate component
   import { Card, section } from '@mantine/core';
   ```

2. **Visual Builder Support**: The Visual Builder now supports Card.Section as a draggable component:
   - Use "Card Section" from the Layout category in the component palette
   - Card.Section should typically be placed inside a Card component
   - Supports `inheritPadding` and `withBorder` props

### Tamagui Projects - CRITICAL IMPORT CONFIGURATION

**‚ö†Ô∏è IMPORTANT**: There is a known issue where Story UI ignores the configured `importPath` and generates imports from `@tamagui/core` or other package paths. This MUST be addressed:

1. **CRITICAL - Import Path Configuration**: 
   - **Problem**: Story UI may generate imports like `import { Button } from '@tamagui/core'` even when configured otherwise
   - **Solution**: Configure your project with local component wrappers and run import fixing scripts
   
   ```javascript
   // ‚ùå WRONG - Story UI may incorrectly generate this
   import { Card, Button, Text } from '@tamagui/core';
   import { AlertDialog } from '@tamagui/alert-dialog';
   
   // ‚úÖ CORRECT - After fixing, imports should be from local components
   import { Card, Button, Text, AlertDialog } from '../../components';
   ```

2. **Required Project Setup for Tamagui**:
   - All Tamagui components should be re-exported through local `/src/components` directory
   - Configure `story-ui.config.js` with `importPath: "../../components"`
   - Run import fixing script after generation: `npm run fix-imports`

3. **Import from the correct package**: When manually writing stories, use `'tamagui'`, NOT `'@tamagui/core'`
   ```javascript
   // ‚úÖ CORRECT for manual stories
   import { Card, Button, Text, View, XStack, YStack } from 'tamagui';
   
   // ‚ùå WRONG - This will cause "does not provide an export" errors
   import { Button } from '@tamagui/core';
   ```

2. **Use valid color tokens**: Tamagui's default config uses specific token naming:
   ```javascript
   // ‚úÖ CORRECT - Use standard tokens from default config
   backgroundColor="$backgroundHover"
   color="$color"
   color="$color10"  // For muted text
   color="$color11"  // For secondary text
   color="$color12"  // For primary text
   
   // ‚ùå WRONG - These tokens may not exist in default config
   backgroundColor="$gray3"
   color="$blue11"
   backgroundColor="$green3"
   ```

3. **Avoid theme props unless confirmed**: Don't assume theme variants exist
   ```javascript
   // ‚úÖ CORRECT - Use default button styling
   <Button>Click me</Button>
   
   // ‚ùå WRONG - theme="blue" may not exist
   <Button theme="blue">Click me</Button>
   ```

4. **Generated stories location**: Stories generated by Story UI will be placed in `src/stories/generated/` directory

5. **NEVER use emoji icons**: Always use Lucide icons from `@tamagui/lucide-icons`
   ```javascript
   // ‚úÖ CORRECT - Use proper Lucide icons
   import { Clock, Users, Star, Heart, ChevronRight } from '@tamagui/lucide-icons';
   
   <Clock size={16} color="$color10" />
   <Users size={16} color="$color10" />
   
   // ‚ùå WRONG - Never use emojis as icons
   <Text>üïí 25 mins</Text>
   <Text>üë• Serves 4</Text>
   <Text>‚≠ê 4.8/5</Text>
   ```

6. **Use Tamagui's Image component**: Always use the Image component from 'tamagui'
   ```javascript
   // ‚úÖ CORRECT
   import { Image } from 'tamagui';
   
   <Image
     source={{
       uri: "https://example.com/image.jpg",
       width: 300,
       height: 200,
     }}
     width="100%"
     height={200}
   />
   
   // ‚ùå WRONG - Don't use HTML img tags
   <img src="image.jpg" />
   ```

## Error Patterns to Avoid

<!-- List common mistakes and how to avoid them -->
1. **Wrong**: `import { Button } from '@tamagui/core'`
   **Right**: `import { Button } from 'tamagui'`
   **Why**: Components are exported from the main 'tamagui' package, not '@tamagui/core'

2. **Wrong**: `backgroundColor="$gray3"` (when using default Tamagui config)
   **Right**: `backgroundColor="$backgroundHover"`
   **Why**: Default Tamagui config may not include all color variants; use standard tokens

3. **Wrong**: `<Button theme="blue">` (unless custom themes are defined)
   **Right**: `<Button>`
   **Why**: Don't assume theme variants exist unless explicitly configured

4. **Wrong**: `<div>...</div>` (in Tamagui projects)
   **Right**: `<View>...</View>`
   **Why**: Use Tamagui's cross-platform components

5. **Wrong**: `style={{margin: '10px'}}`
   **Right**: `margin="$4"` or `padding="$4"`
   **Why**: Use Tamagui's token system for consistent spacing

6. **Wrong**: `<Text>üïí 25 mins</Text>`
   **Right**: `<XStack gap="$1"><Clock size={16} /><Text>25 mins</Text></XStack>`
   **Why**: Emojis are not accessible, scalable, or themeable - use proper icon components

7. **Wrong**: `<img src="photo.jpg" alt="Recipe" />`
   **Right**: `<Image source={{ uri: "photo.jpg" }} alt="Recipe" />`
   **Why**: Use Tamagui's Image component for proper cross-platform support
